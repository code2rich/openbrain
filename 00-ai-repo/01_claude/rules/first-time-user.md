# 初始化用户友好规则

## 核心原则

系统的每个功能都必须考虑首次使用的初始化用户。任何人 `git clone` 后应该能在最短时间内跑起来并完成所有配置。

## 具体要求

### 零配置启动

- `docker compose up` 必须能直接跑起来，不依赖预置文件
- 缺失的运行时文件（index.md、log.md、config.json 等）必须有 graceful 降级
- 缺文件不报错、不阻断启动，只在需要时提示用户配置

### UI 配置闭环

- 所有配置（API Key、模型选择、供应商等）应能在页面 UI 完成
- 不依赖手动编辑文件或设置环境变量
- 设置页面要有清晰的引导（placeholder、提示文案、测试连接按钮）

### 模板文件规范

- 每个需要用户配置的文件必须有对应的 `.example` 模板
- 模板文件必须注释清晰，说明每个字段的含义和可选值
- 模板中的占位值要明显是占位符（如 `your-api-key-here`），不能误导用户以为已配置好

### 文档覆盖

- README 必须包含完整的初始化步骤
- 初始化步骤应该从零开始，不假设用户有任何前置知识
