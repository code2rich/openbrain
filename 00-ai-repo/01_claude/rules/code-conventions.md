# 代码与工程规则

## 安全

- 不引入命令注入、XSS、SQL 注入等 OWASP Top 10 漏洞
- 发现不安全代码立即修复
- 只在系统边界验证（用户输入、外部 API），内部代码信任框架保证

## 简洁

- 三行相似代码 > 一个过早抽象
- 不为一次性操作创建辅助函数或抽象
- 不为假设的未来需求设计
- 不添加错误处理、降级或验证不可能发生的场景

## 修改

- 不添加未经请求的功能、重构或改进
- Bug 修复不需要清理周围代码
- 简单功能不需要额外可配置性
- 不在未修改的代码上添加文档字符串、注释或类型注解

## Git

- 只在用户明确要求时创建 commit
- 不要 force push 到 main/master
- 优先创建新 commit 而非 amend
- 不要跳过 hooks（--no-verify）
