# 安全策略规则

## 漏洞报告

- 安全漏洞不应在公开 Issue 中报告
- 通过 GitHub Security Advisories 或私有邮件报告
- 收到报告后 48 小时内确认，7 天内评估，30 天内修复

## 代码安全

- 所有外部输入必须验证和清理（文件上传、API 参数、用户输入）
- 文件路径操作必须防止目录遍历攻击
- API 端点必须有适当的错误处理，不泄露内部堆栈信息
- 依赖更新：定期运行 `npm audit`，修复 Critical/High 级别漏洞

## 数据安全

- 用户数据（00-raw/）不纳入版本控制
- API Key 等凭证通过环境变量或加密配置文件管理
- Docker 容器以非 root 用户运行（生产环境）
- 挂载卷使用最小权限原则（只读挂载 wherever possible）

## 合规

- 第三方代码必须标注 License
- 不引入 GPL 等传染性协议的依赖（项目为 MIT）
- AI 生成的代码需经过人工审查后才能合并
