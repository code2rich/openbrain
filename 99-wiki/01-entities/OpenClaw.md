## 本质定位

OpenClaw 是一整套 AI Agent 的**操作系统**，而非单一 Agent 工具。核心思路是通过中央 Gateway 网关连接外部通信渠道，分配任务给对应 Agent 执行。

核心公式：

```
OpenClaw = Agent + 守护进程 + channel适配 + 记忆系统 + 定时任务 + 心跳检测 + 工具集成 + 安全框架
```

底层基于标准 Agent 执行循环，调用 Claude API，在此基础上完成各类自动化任务。

## 核心特色

| 特色 | 说明 |
|------|------|
| 多渠道入口 | 50+ 消息平台开箱可用（钉钉、微信、Telegram 等） |
| 跨会话持久记忆 | 日记忆 + 长记忆，向量检索 + BM25 混合检索 |
| 主动调度 | 心跳检测 + 定时任务，Agent 可主动触发 |
| 工具生态 | 高开放度工具集成，任务全自动执行 |
| 全开源 | 支持私有部署 + 本地模型（vLLM） |
| 社区生态 | ClawHub 公共 Skills 注册中心，三个月 0→30W+ GitHub Star |

## 架构设计

### 分层架构

```
控制接口层 → 网关层（Gateway） → 智能体运行时环境 → 工具与能力层
```

关键设计：**调度中心架构**，消息经中央 Gateway 分配至正确 Agent，消息通信/接口层与 Agent 思考执行逻辑分离。

### 消息处理流程

```
会话路由 → 上下文组装 → 模型调用 → 技能选择与工具执行 → 格式化返回
```

核心循环：接收 → 上下文组装 → 模型推理 → 工具执行 → 流式回复 → 持久化

### 核心组件

- **网关（Gateway）**：拥有所有会话状态，远程模式下会话存储于网关主机
- **记忆系统**：日记忆 + 长记忆，向量检索 + BM25 混合检索

### 扩展能力

四个维度可扩展：渠道扩展、记忆扩展、工具扩展、多渠道模型对接

## 应用场景

### 动态/非固定流程

原子能力随机组合：新闻摘要、技术报告、定时提醒、邮件管理、日程安排、网页浏览、文件读写、Shell 命令执行、代码编写、数据查询分析。

### 固定业务流程

标准化流程固化：

- **投资情报引擎**：4-Agent 情报网络，7×24 运行
- **产品管理**：4-Agent PM 运营网络，每周节省 20+ 小时
- **企业通用**：CRM 跨系统对账、发票/凭证同步、Jira/工单同步 IM、监控告警转 IM+建工单

### 恒生定制场景

多层级落地架构：端建设 → 中间层（接口控制层、智能体层、工具技能层）→ 底层资源

## 应用构建方法

### 三步法

1. **分析业务流程**：拆解重复性任务为核心步骤
2. **构建工具与技能**：
   - Tools（MCP 服务）：定义工具清单 → 描述工具 + 定义出入参 → 实现逻辑
   - Skills（技能包）：定义名称 → 创建 SKILL.md → 准备参考附件
3. **调试与优化**：MCP 工具配置 + Skill 技能配置

### 实战案例

- 自动化内容审核（封装效能质量审核 Skill）
- 自动盯盘（自定义 stock-monitor Skill）
- 技术洞察生成（调用 Brave Search、文件操作等工具）
- 数据库查询与钉钉卡片接入
- 自动数据填报（搭建可配置 Agent 平台）

## 安全与运维

### 安全风险

- 社交工程诱导、操作失控
- 防控：权限最小化、漏洞扫描、禁止私自在公司内部署

### 常见问题

- Gateway 卡死、僵尸进程霸占端口、进程抢端口、连接重新配对
- 解决思路：启动双服务实例，相互检测、相互修复

## 社区生态

- **ClawHub**：公共 Skills 注册中心
- 操作方式：网页浏览 Skills，CLI 完成搜索/安装/更新/发布
- 核心命令：`clawhub install XXX`
