# 团队管理方法论

## 三三制团队管理

编组三三、队形三角、角色互补与能力备份。

### 落地四步

1. 摸排业务模块
2. 人员能力摸底
3. 模块职责划分（主责、备份责任人、掌握技能人）
4. 建立培养计划

### 关键要求

- 所有研发都要会使用 AI 工具
- 后端要开始会用 AI 工具写前端
- 前端要么成为前端专家，要么也会写后端
- 数据算法部分成员也需会写后端 Java 代码

## 研发部长六项角色

1. 质量守护
2. 效率提升
3. 团队能力
4. 版本管理
5. 产品规划
6. 竞争力落地

## 高绩效团队三维画像

- **业务结果**：交付质量、交付效率
- **团队氛围**：信任、协作、开放
- **团队能力**：技术能力、业务能力、学习能力

### 绩效飞轮

习惯 + 意愿 + 能力 → 定目标 / 追过程 / 拿结果 → 高绩效

## 质量管理框架

### 正向设计（破除四拍）

- 四拍：拍脑袋决策、拍胸脯保证、拍大腿后悔、拍屁股走人
- 破除方法：数据驱动决策、量化承诺、过程监控、结果闭环

### 全链路质量基石

需求 → 设计 → 开发 → 测试 → 发布

### 复盘四步

1. 回顾目标
2. 评估结果
3. 分析原因
4. 总结规律

## 效率提升"二五三三"框架

- 两个转变
- 五个策略
- 三个意识
- 三个坚持

## IDP 四阶 7 步法

1. 明确目标
2. 制定计划
3. 检验合理性
4. 持续推动落地

## 角色认知转变

| 从 | 到 |
|----|-----|
| 先交活 | 先搭台 |
| 看个人 | 看结构 |
| 凭感觉 | 看数据 |
| 派活的 | 搭班子的 |
