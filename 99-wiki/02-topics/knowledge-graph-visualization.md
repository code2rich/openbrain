# 个人知识图谱可视化

## 方法论：知识蒸馏 → 图谱构建 → 可视化

将个人历史文章转化为可视化知识图谱的三步流程：

1. **知识蒸馏**：用 LLM 逐篇提取知识节点、关系、摘要，输出结构化三元组文件
2. **增量合并**：逐文件读取，去重合并节点和边，权重累加，形成完整图谱
3. **可视化渲染**：ECharts 力导向图展示，支持拖拽、缩放、节点交互

## 技术栈

| 组件 | 选型 | 理由 |
|------|------|------|
| 前端框架 | React 18 + TypeScript | 类型安全，组件化 |
| 构建工具 | Vite | 快速热更新 |
| 图表库 | ECharts 5（力导向图） | 成熟的 graph 类型，支持力引导布局 |
| 样式 | Tailwind CSS | 原子化 CSS |
| 文件读取 | File System Access API | 浏览器原生目录选择 |
| LLM API | DeepSeek（OpenAI 兼容格式） | 低成本，temperature=0.3 确保稳定输出 |

## 核心数据模型

```
GraphNode: id + label + category + weight + sources[]
GraphEdge: source + target + relation + weight + articleIds[]
```

关键设计：
- 节点 id 使用中文概念词（如"实践"、"归纳"），全局唯一
- 边去重基于 source+target+relation 三元组
- 权重按出现次数累加，影响节点大小和标签显示

## 图谱交互设计

```
┌────────────────────────────────────────────────────────────┐
│ TOOLBAR: 进度条 + 状态文字 + [读取] [重置]                  │
├──────────────────────────────────┬─────────────────────────┤
│                                  │ 节点详情（右上）         │
│     力导向图                     │ - 概念名 + 分类          │
│     - 按分类着色                 │ - 出现文章数             │
│     - 权重决定节点大小           │ - 各文章摘要             │
│     - 点击高亮相邻节点          ├─────────────────────────┤
│     - 节点可拖动                 │ 相关文章（右下）         │
│                                  │ - 按相邻节点分组         │
│                                  │ - 关系路径展示           │
└──────────────────────────────────┴─────────────────────────┘
```

节点分类颜色方案：思维类（蓝）、个人成长类（青绿）、管理类（琥珀）、数字化类（橙）。

## LLM 知识蒸馏 Prompt 要点

- System Prompt 约束：只返回纯 JSON，只提取新增节点，每篇最多 15 节点 + 20 条关系
- User Prompt 动态注入已有节点列表，避免重复
- 低 temperature（0.3）确保输出格式稳定

## 性能策略

- 每文件处理后 sleep 500ms，防止 API 限流
- ECharts 使用 `setOption({ notMerge: false })` 增量更新
- 节点超过 200 个时，隐藏低权重节点标签（weight < 3）

## 实践案例

作者何明璐（人月聊IT）将个人公众号历史文章构建为知识图谱，四大知识领域：

| 领域 | 颜色 | 覆盖内容 |
|------|------|---------|
| 思维类 | 蓝 | 认知方法论、逻辑思维 |
| 个人成长类 | 青绿 | 技能发展、学习策略 |
| 管理类 | 琥珀 | 项目管理、团队管理 |
| 数字化类 | 橙 | 企业架构、技术转型 |

第一版问题：展开到最细子节点导致图谱过于复杂。优化方案：底层节点默认收缩，点击后展开。
