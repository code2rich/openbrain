# Claude Code 研发实战培训

## 课程结构（2 小时 · 7 部分）

| 部分 | 主题 | 时间占比 | 核心内容 |
|------|------|---------|---------|
| 一 | 痛点洞察与机遇 | 10% | 传统流程 → AI 提效机会表 → 代码评审困境 |
| 二 | 产品设计与迭代 | 20% | MVP → 通用 → 好用（三板斧演进） |
| 三 | 技术选型与架构 | 10% | 工具/模型/语言/架构/存储选型 |
| 四 | Claude Code 完全指南 | 15% | 六大优势 + 常用命令 + MCP/Skills/Rules |
| 五 | 提示词与最佳实践 | 10% | SMART 原则 + 六大技巧 + 十大陷阱 |
| 六 | 核心要点回顾 | 5% | 产品思路 + 技术路径 + 工具使用 |
| 七 | 实战演练 | 30% | 需求收集 → MVP 设计 → 工具落地 |

## 思想重塑：从"副驾驶"到"承包商"

- 传统 AI 辅助：人主导，AI 建议
- Claude Code 模式：AI 承包任务，人审查结果
- 异步执行解放高级工程师，专注架构设计

## Workspace 三层体系

| 层级 | 文件 | 作用 |
|------|------|------|
| 规范层 | CLAUDE.md | 构建命令/目录结构/代码规范/禁止操作 |
| 能力层 | Skills | 可复用的专业能力包 |
| 连接层 | MCP | 外部工具集成 |

## 提示词工程黄金公式

清晰角色 + 具体任务 + 上下文约束 + 输出格式 + 示例

### SMART 原则

| 维度 | 含义 | 提示词要求 |
|------|------|------------|
| S (Specific) | 明确 | 指定技术栈、说明使用场景 |
| M (Measurable) | 可测量 | 明确成功标准、性能要求 |
| A (Achievable) | 可实现 | 符合项目规模 |
| R (Relevant) | 相关 | 遵循团队规范 |
| T (Time-bound) | 有时限 | 明确优先级、交付时间 |

## 上下文工程三层架构

1. **持久层**：CLAUDE.md、项目文档（长期不变）
2. **会话层**：对话历史、中间产物（会话内有效）
3. **实时层**：当前代码、报错信息（即时上下文）

## 三种编程范式对比

| 维度 | Vibe Coding | SDD | CDD |
|------|-------------|-----|-----|
| 核心资产 | 聊天记录（易失） | 结构化 Spec（持久） | 活上下文（持续演化） |
| 入口指令 | 任意自然语言 | 人写 PRD | AI 自动生成规格 |
| 存量项目支持 | 弱 | 中 | **强**（自动逆向） |
| upfront 时间 | 几乎 0 | 高（人写规格） | 中（AI 生成，人审核） |
| 最佳场景 | 快速原型、单人 MVP | 新功能、需合规覆盖 | 祖传代码库、架构复杂 |

## 工具评级

- **S 级**：Claude Code（纯命令行/200k 上下文/零数据留存/超长会话）
- **A 级**：Cursor（AI 原生 IDE，VS Code 生态）
- **B+ 级**：Kiro（规范驱动，Agent Hooks）
- **C 级**：Windsurf（Cascade 引擎，近期不支持 Claude 4）

## 实战案例：KypCodeReview

- 68,000 行代码，100% AI 生成
- 7 月 MVP（1 周）→ 9 月全自动化（1 周）→ 持续迭代
- 技术栈：Flask + Bootstrap 5 + DeepSeek/OpenAI API + GitPython
- 产品三阶段：MVP（手动选择提交检视）→ 通用（多项目多模型）→ 好用（全托管自动化）

## 十大常见陷阱

| 陷阱 | 核心原则 | 记忆口诀 |
|------|----------|----------|
| 过度信任 | 验证优先 | AI 生成，我来验证 |
| 上下文不足 | 充分说明 | 背景讲清楚，AI 才靠谱 |
| 忽略审查 | 必须审查 | 代码不审查，上线两行泪 |
| 任务过大 | 拆分执行 | 大化小，小化了 |
| 缺少测试 | 测试驱动 | 先写测试，再写代码 |
| 忽略安全 | 安全第一 | SQL 注入、XSS、硬编码密钥 |
| 不用版本控制 | Git 保护 | 提交前先 diff |
| 忽略性能 | 性能意识 | 警惕 N+1 查询 |
| 过度依赖 | 保持判断 | AI 是助手，不是拐杖 |
| 忽视团队 | 规范统一 | CLAUDE.md 约束 AI 行为 |

## 决策树

- 任务明确 + 可异步 → Claude Code
- 需实时反馈 → Cursor / Copilot

## 落地路径

1. 5 分钟快速启动：最小化 CLAUDE.md + 3 个 Skill
2. 持续迭代：错误率 30% → 5%
3. 验收标准：能跑通 / 解决痛点 / 可量化

## 课后作业

1. **需求分析**：找 3 个 AI 提效场景（含使用前耗时估算）
2. **工具开发**：用 Claude Code 开发一个提效工具（能跑通 + 解决痛点 + 可量化）
3. **经验总结**：至少 3 条技巧 + 踩坑 + 思考（≥500 字）
